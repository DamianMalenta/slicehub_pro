# Nowy projekt — polecenia przez pliki `.md`

Ten katalog to **prosty tor**: Ty (albo PM) **piszecie polecenia i ustalenia w plikach markdown** w jednym folderze projektu; zespół i Cursor pracują **w oparciu o te pliki** — zamiast rozmów tylko w czacie.

- **[INSTRUKCJA_START_OD_ZERA.md](INSTRUKCJA_START_OD_ZERA.md)** — jak to zorganizować od zera.
- **[OCZEKIWANIE_NA_UZGODNIENIA_ZESPOLU.md](OCZEKIWANIE_NA_UZGODNIENIA_ZESPOLU.md)** — co doprecyzować z ludźmi (role, kto edytuje które `.md`).
- **[tools/bootstrap_nowy_projekt.ps1](tools/bootstrap_nowy_projekt.ps1)** — opcjonalnie: utworzy pusty szkielet folderów + krótkie README na dysku docelowym.

**Tu nie ma żadnego osobnego panelu ani automatycznego wpychania tekstu do Cursor** — Ty piszesz `.md`, zespół je czyta i realizuje (w Cursorze przez `@plik.md` jak zwykle).
