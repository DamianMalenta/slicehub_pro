# Uzgodnienia zespołu — praca przez `.md`

Ten dokument uzupełnicie **z ludźmi**, jak tylko ustalicie sposób pracy. Sama technika „pliki markdown z poleceniami” jest prosta i **nie wymaga** żadnego dodatkowego oprogramowania poza Git + edytorem.

## Pytania robocze

- [ ] Gdzie leży **główny plik celu** (nazwa: `BRIEF.md`, `CEL.md`, …)?
- [ ] Kto może **zmieniać** polecenia — tylko Ty, czy też lead?
- [ ] Czy **`KOLEJKA.md`** (lub tablica issue) jest obowiązkowa przy większym zespole?
- [ ] Jak oznaczacie **„done”** — checkbox w `.md`, PR, coś trzeciego?

Po ustaleniach dopisz tu 10–15 linii „jak pracujemy” i link w głównym README projektu docelowego.
